# Doggo Dashboard v1.0 功能摘要

這份文件是 `v1.0` 版本的簡短功能整理，方便快速理解這個版本目前包含什麼。

## 版本定位

`v1.0` 是目前已確認可用的穩定版基準點：
- dashboard 視覺已回退到霓虹 cyberpunk 大改版之前
- `dog.koxuan.com` 已成功跑在 VPS 上
- GitHub Actions 已可自動部署到 VPS

## 目前包含的功能

### 1. Dashboard 主介面
- 像素風狗狗主題儀表板
- 日夜主題切換
- 主舞台 + 情報卡片式版面
- 靜態前端展示頁

### 2. 市場與內容資料
- 台股快報
- 新聞雷達
- 川普貼文摘要區塊
- 任務 / 狀態類資訊卡
- 天氣卡片
- 便宜機票觀察資料

### 3. 天氣區塊
目前天氣資料使用三個地點：
- 石牌
- 中和
- 松山

其中 commute 建議文案目前是寫死用：
- **中和 → 松山**

這代表它目前不是「三地任意路線智慧判斷」，而是：
- 固定拿 `zhonghe` 與 `songshan` 這兩點做通勤提醒
- 只是在同一張 weather 卡裡，同時展示石牌 / 中和 / 松山 三個地點資料

所以如果每次都看到：
- 「中和到松山這段目前看起來還算穩，先不用太擔心下雨。」

它的真正意義是：
- dashboard 目前內建了一條固定通勤路線提醒
- 並不是在根據你當下真的要去哪裡動態切換

### 4. 部署與站點
- `dog.koxuan.com` 由 VPS 提供
- Caddy 負責 HTTPS 與靜態站服務
- 站點目錄：`/srv/www/dog.koxuan.com/current`
- GitHub Actions push 後可自動部署

### 5. 維運能力
- 可手動部署：`scripts/deploy_dog_dashboard.sh`
- 可自動部署：GitHub Actions → VPS
- 已建立安全版 deploy user：`dogdeploy`
- 已有 `v1.0` tag 可回退

## 目前已知限制

- 天氣 commute 提醒目前是固定路線邏輯，不是通用型邏輯
- Node 20 warning 尚未清理，但不影響現在部署
- rollback 流程還沒有做成正式版本管理機制

## 建議後續方向

1. 把 weather commute 提醒改成可配置路線
2. 增加 deployment rollback 機制
3. 規範更多子站點部署模式
4. 視需要再重做更成熟的視覺升級，而不是直接走過重的霓虹風
