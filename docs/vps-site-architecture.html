我看到的 VPS 現況

基礎不差
Ubuntu 24.04、UFW 開啟、只放行 22/80/443、Fail2ban 有開、Caddy 正常、Tailscale 正常。
仍要注意
SSH 目前允許 root 直接登入，雖然已禁用密碼登入，但長期來說還是建議再收斂。
  • 適合當你未來多站點的入口主機
  • 不適合每個服務自己亂開 port 對外
  • 最好的做法是所有外部流量都先進 Caddy

我建議的未來架構

把這台 VPS 變成你的網站入口主機，所有站都走 subdomain，靜態站跟動態服務分流管理。

Internet
   |
   |-- dog.koxuan.com ----> VPS :80/:443
   |-- bot.koxuan.com ----> VPS :80/:443
   |-- lab.koxuan.com ----> VPS :80/:443
   |-- api.koxuan.com ----> VPS :80/:443
   |
   v
+------------------------------------------------+
| VPS 139.59.122.96                              |
|                                                |
|  Caddy (唯一公開入口)                          |
|                                                |
|  路由規則：                                     |
|  - dog.koxuan.com -> 靜態網站                  |
|  - bot.koxuan.com -> reverse proxy             |
|  - lab.koxuan.com -> 靜態網站 / 測試站         |
|  - api.koxuan.com -> 後端服務                  |
+----------------------+-------------------------+
                       |
        +--------------+--------------+
        |                             |
        v                             v
  /srv/www/...                  127.0.0.1 / Tailscale / Docker
  靜態網站目錄                    動態服務 / 私網服務 / API

這種設計的核心邏輯

  • Caddy 是唯一公開入口，TLS 與 hostname 都交給它
  • 靜態站 直接由 VPS 讀檔提供
  • 動態站 用 reverse proxy 指到本機服務、Docker 或 Tailscale
  • 每站一個 subdomain，未來加網站不需要重做架構
乾淨 可擴充 好維護 安全面較好控

Caddy 概念

dog.koxuan.com {
    root * /srv/www/dog.koxuan.com/current
    file_server
}

bot.koxuan.com {
    reverse_proxy 100.74.22.60:18889
}

我建議的分階段做法

保留 bot.koxuan.com 現況
先不要動已經在跑的反代服務，避免影響現有入口。
新增 dog.koxuan.com 到 VPS
把 dashboard 靜態檔部署到 VPS，讓 Caddy 直接 serve。
整理部署流程
後續再做 deploy script、自動同步、rollback 與備份。
之後再做安全收斂
例如非 root 部署、SSH root login 收緊、站點管理規範化。
一句話總結
這台 VPS 最適合被設計成你的 網站入口主機。
靜態站直接由 Caddy 提供，動態服務走 reverse proxy，每個網站各自一個 subdomain，之後你要加更多站才不會亂掉。
